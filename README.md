# learn-github

PMO ย่อมาจาก Project Management Office หรือ "สำนักงานบริหารโครงการ" คือหน่วยงานกลางที่จัดตั้งขึ้นเพื่อกำหนดมาตรฐาน วางระบบกระบวนการทำงาน สนับสนุน และกำกับดูแลโครงการต่างๆ ในองค์กรให้ดำเนินไปอย่างถูกต้อง มีประสิทธิภาพ ทันเวลา และสอดคล้องกับกลยุทธ์ทางธุรกิจหลัก เพื่อให้โครงการประสบความสำเร็จตามเป้าหมาย

หน้าที่หลักของ PMO (Project Management Office)
กำหนดมาตรฐาน (Governance): สร้างมาตรฐาน เครื่องมือ (Tools) เทมเพลต (Templates) และระเบียบปฏิบัติ (Methodologies) ในการบริหารโครงการให้เป็นรูปแบบเดียวกัน
สนับสนุนโครงการ (Support): ให้คำปรึกษา ฝึกอบรม และสนับสนุนทีมผู้จัดการโครงการ (PM) ด้านการบริหารจัดการทรัพยากรและแก้ไขปัญหา
บริหารจัดการพอร์ตโฟลิโอ (Portfolio Management): จัดลำดับความสำคัญของโครงการ (Prioritize) และจัดสรรทรัพยากร (คน/งบประมาณ) อย่างเหมาะสม
ติดตามและรายงานผล (Monitoring & Reporting): ติดตามความคืบหน้า รายงานผลการดำเนินงาน และประเมินความเสี่ยงต่อผู้บริหารระดับสูง
เก็บข้อมูลและบทเรียน (Knowledge Management): รวบรวมข้อมูล, บทเรียนที่ได้รับ (Lessons Learned) และสรุปแนวปฏิบัติที่ดีที่สุด (Best Practices) เพื่อนำไปพัฒนาโครงการต่อไป
